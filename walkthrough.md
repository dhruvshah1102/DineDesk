# MenuFlow - Multi-tenant SaaS Walkthrough

MenuFlow is a comprehensive Next.js 14 based B2B2C application designed to provide cafes with digital menus, robust order management Kanban boards, and integrated payments.

## Architecture and Tenancy

The app natively utilizes **Subdomain Routing** handled at the edge via [middleware.ts](file:///c:/Lucide%20tECH/petpooja/src/middleware.ts). 

- **Super Admin**: `admin.menuflow.com` -> `src/app/(admin)`
- **Cafe Owners**: `app.menuflow.com` -> `src/app/(dashboard)`
- **Customers**: `{slug}.menuflow.com` -> `src/app/(customer)`

### Local Dev Testing
Since local domains don't easily support subdomains natively without proxy mapping (like `localtunnel` or `hosts` file hacks), the `middleware.ts` automatically proxies localhost testing using the `?cafe={slug}` URL query parameters.
- E.g, navigate to `http://localhost:3000/?cafe=brew-house` which acts as `brew-house.menuflow.com` to act as a customer.
- Navigate to `http://localhost:3000/?cafe=app` to access the Cafe Owner Dashboard login.
- Navigate to `http://localhost:3000/?cafe=admin` to access the Super Admin Dashboard login.

## Features Implemented

### 1. Advanced Real-time Integrations
- Customer places an order via dynamic cart interaction.
- The Owner's Kanban dashboard receives the payload instantaneously without refreshing using `Supabase Realtime PostgreSQL subscriptions`.
- Status updates (`New` -> `Preparing` -> `Ready`) immediately notify the Customer's isolated tracking screen.

### 2. High-Fidelity UI/UX Components
- Built out leveraging custom Tailwind configurations based around centralized design tokens (e.g., `--brand-color`, `var(--customer-brand)`).
- Cart system implements `localStorage` synchronized state ensuring selections are saved per specific table interactions.

### 3. Integrated Payment Architecture
- Cart passes checkout to public API generating genuine `Razorpay Orders`.
- Frontend utilizes the Razorpay script allowing popup UI checkouts with auto pre-fill of cafe contexts correctly scoped to individual Tenants.
- Server validates cryptographic HMAC signatures directly generated by Razorpay prior to database status confirmation preventing malicious network modifications.

## Next Steps to Execute

1. **Verify Services**: Ensure you have executed `npx prisma generate` and `npx prisma db push` (or migrate) along with `npx prisma db seed` to insert the default **SuperAdmin** and **"The Brew House"** cafe mock data.
2. **Start the App**: Run `npm run dev` to boot the Next.js server.
3. **Log In**: 
   - Open `localhost:3000/?cafe=admin` -> Log in using `super@menuflow.com` | `admin123`
   - Open `localhost:3000/?cafe=app` -> Log in using `owner@brewhouse.com` | `password123`
4. **Order Live**: Use an incognito tab pointing to `localhost:3000/table/1?cafe=brew-house` and watch the owner Kanban jump to life when you checkout!
